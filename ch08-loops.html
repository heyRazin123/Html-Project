8.1 — ConceptWhat are Loops?

A loop is a control structure that repeats a block of code multiple times. Without loops, if you wanted to print "Hello" 1,000 times, you'd need 1,000 lines of code. With a loop, you need 3 lines. Loops are the foundation of automation.

Real-World Analogy — Assembly Line

Imagine a car factory. The same set of steps (weld frame → add engine → paint) is performed repeatedly on each car that comes down the assembly line. Nobody rewrites the instructions for each car. A loop is your assembly line — define the steps once, execute them for as long as needed.

Java has three loop constructs: for, while, and do-while. They are all equally powerful — any loop can be written with any of the three. The choice depends on which reads most naturally for your use case.

🔢
8.2 — Loop 1The for Loop

Use the for loop when you know in advance how many times the loop should run. It is the most compact loop — init, condition, and update are all in one line.

Anatomy of a for Loop

for Loop — Full Breakdown
// Syntax: for (initialization; condition; update) { body }
for (int i = 1; i <= 5; i++) {
    System.out.println("i = " + i);
}

// ↑ Breakdown:
// int i = 1  → initialization: runs ONCE before loop starts
// i <= 5     → condition:      checked BEFORE each iteration
// i++        → update:         runs AFTER each iteration body

Execution Flow — Step by Step

for (int i = 1; i <= 5; i++) { println(i); } ┌─────────────────────────────────────────────────────┐ │ Step 1: Initialize → int i = 1 (runs ONCE only) │ │ Step 2: Check → i <= 5? YES │ │ Step 3: Body → println(1) │ │ Step 4: Update → i++ → i = 2 │ │ Step 5: Check → i <= 5? YES │ │ Step 6: Body → println(2) │ │ Step 7: Update → i++ → i = 3 │ │ ... (same for 3, 4, 5) │ │ Step N: Check → i <= 5? i=6 → NO → EXIT │ └─────────────────────────────────────────────────────┘ Order: Init → [Check → Body → Update] → [Check → Body → Update] → ... → Check(false) → Done

Common for Loop Variations

for Loop Variations
// Count UP (1 to n):
for (int i = 1; i <= 10; i++) { ... }

// Count DOWN (n to 1):
for (int i = 10; i >= 1; i--) { ... }

// Count by 2s (evens: 0,2,4,6,8):
for (int i = 0; i < 10; i += 2) { ... }

// Count by 5s:
for (int i = 0; i <= 100; i += 5) { ... }

// Index starts at 0 (array traversal):
for (int i = 0; i < arr.length; i++) { ... }

// Multiple variables (rare but valid):
for (int i = 0, j = 10; i < j; i++, j--) { ... }

// Infinite loop (use with care):
for (;;) { /* runs forever — needs break inside */ }

// Classic: Sum 1 to N
int sum = 0;
for (int i = 1; i <= 100; i++) sum += i;
System.out.println("Sum = " + sum);  // 5050

8.3 — Loop 2The while Loop

Use while when you don't know in advance how many times the loop should run — it depends on a condition that changes during execution. The condition is checked before each iteration.

while Loop Syntax and Flow

while Loop
// Syntax:
while (condition) {
    // body — runs as long as condition is true
    // MUST eventually make condition false (or use break)
}

// Example 1: Same as for loop (count 1-5)
int i = 1;            // initialization (outside)
while (i <= 5) {       // condition
    System.out.println(i);
    i++;               // update (inside body)
}

// Example 2: Digit extraction (count unknown)
int n = 12345;
int count = 0;
while (n > 0) {
    count++;
    n /= 10;   // remove last digit
}
System.out.println("Digits: " + count);  // 5

// Example 3: Read until sentinel (-1)
Scanner sc = new Scanner(System.in);
int total = 0, num;
System.out.println("Enter numbers (-1 to stop):");
while ((num = sc.nextInt()) != -1) {
    total += num;
}
System.out.println("Total: " + total);
while FLOW: ┌──────────────────────────────┐ │ │ │ ┌──────────────────────┐ │ │ │ Check condition │ │ │ └──────────┬───────────┘ │ │ │ │ │ ┌────┴────┐ │ │ │ │ │ │ TRUE FALSE ──────►│ (exit loop) │ │ │ │ ▼ │ │ ┌──────────────────────┐ │ │ │ Execute body │ │ │ └──────────┬───────────┘ │ │ │ │ │ └───────────────┘ (back to check) KEY: Condition checked BEFORE body. May run 0 times if condition starts false.

while vs for — When to Choose

Use for when:
  • Number of iterations known in advance
  • Iterating over a range (1 to n)
  • Iterating over an array
  • Counter-based loops

Example: Print multiplication table — you know it runs exactly n times.

Use while when:
  • Number of iterations NOT known in advance
  • Reading input until user stops
  • Processing until condition changes
  • Game loops, menu loops

Example: Read numbers until -1 — you don't know how many numbers the user will enter.

8.4 — Loop 3The do-while Loop

The do-while loop is like a while loop, but with one critical difference: it checks the condition after the body runs. This guarantees the body executes at least once, regardless of whether the condition is true or false.

do-while Loop
// Syntax: (note semicolon at end!)
do {
    // body — runs at LEAST once
} while (condition);  // ← semicolon required here

// Example 1: Always runs once even if condition is false
int x = 100;
do {
    System.out.println("This runs once!");  // prints despite x > 5
} while (x < 5);  // false from the start — but body ran once

// Example 2: Input validation — BEST use case for do-while
Scanner sc = new Scanner(System.in);
int choice;
do {
    System.out.print("Enter a number between 1-10: ");
    choice = sc.nextInt();
    if (choice < 1 || choice > 10)
        System.out.println("Invalid! Try again.");
} while (choice < 1 || choice > 10);
System.out.println("You entered: " + choice);

// Example 3: Menu-driven program
int option;
do {
    System.out.println("1. Play  2. Settings  3. Quit");
    option = sc.nextInt();
    switch (option) {
        case 1: System.out.println("Playing..."); break;
        case 2: System.out.println("Settings..."); break;
        case 3: System.out.println("Goodbye!"); break;
        default: System.out.println("Invalid choice");
    }
} while (option != 3);

do-while: Check After, Not Before

do-while FLOW: ┌──────────────────────────────┐ │ │ │ ┌──────────────────────┐ │ │ │ Execute body ←───────┼───┘ (FIRST run — no check) │ └──────────┬───────────┘ │ │ │ ┌──────────▼───────────┐ │ │ Check condition │ │ └──────────┬───────────┘ │ │ │ ┌────┴────┐ │ │ │ │ TRUE FALSE ──────► (exit loop) │ │ │ └─────────────────────► (back to body) KEY: Body always runs at least ONCE. Use case: show menu at least once, validate input at least once.

8.6 — AdvancedNested Loops

A nested loop is a loop inside another loop. The inner loop runs completely for every single iteration of the outer loop. This creates a grid-like pattern of execution.

Nested Loops — Multiplication Table
// 3x3 multiplication table
for (int i = 1; i <= 3; i++) {        // outer: 3 rows
    for (int j = 1; j <= 3; j++) {    // inner: 3 columns per row
        System.out.printf("%4d", i * j);
    }
    System.out.println();               // new line after each row
}
// Output:
//    1   2   3
//    2   4   6
//    3   6   9
//
// Total iterations: 3 × 3 = 9

How Nested Loops Execute

for (i=1; i<=3; i++) { ← OUTER LOOP for (j=1; j<=3; j++) { ← INNER LOOP print(i*j); } println(); } Execution trace: i=1: j runs 1→2→3 (inner completes fully) then newline i=2: j runs 1→2→3 (inner completes fully) then newline i=3: j runs 1→2→3 (inner completes fully) then newline Rule: Inner loop = outer_iterations × inner_iterations total runs Here: 3 × 3 = 9 total print(i*j) calls
Nested Loop — Right Triangle Pattern
int n = 5;
for (int i = 1; i <= n; i++) {          // outer: row number
    for (int j = 1; j <= i; j++) {      // inner: row i has i stars
        System.out.print("* ");
    }
    System.out.println();
}
// Output:
// *
// * *
// * * *
// * * * *
// * * * * *

8.7 — Controlbreak and continue

break — Exit the Loop Immediately

break immediately exits the innermost enclosing loop (or switch). Execution continues with the statement after the loop. Used when you've found what you need and want to stop early.

break — Exit Loop Early
// Example 1: Find first number divisible by 7 between 1-50
for (int i = 1; i <= 50; i++) {
    if (i % 7 == 0) {
        System.out.println("First: " + i);  // prints 7
        break;  // stop — no need to continue
    }
}

// Example 2: Linear search with break
int[] arr = {5, 12, 8, 3, 19, 7};
int target = 8, foundAt = -1;
for (int i = 0; i < arr.length; i++) {
    if (arr[i] == target) {
        foundAt = i;
        break;  // found it — stop searching
    }
}
System.out.println(foundAt == -1 ? "Not found" : "Found at index " + foundAt);
// Found at index 2

continue — Skip to Next Iteration

continue skips the rest of the current iteration and jumps directly to the update step (for loop) or back to the condition check (while/do-while). The loop does NOT exit — it continues with the next iteration.

continue — Skip Current Iteration
// Example 1: Print only odd numbers 1-10
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) continue;  // skip even numbers
    System.out.print(i + " ");
}
// Output: 1 3 5 7 9

// Example 2: Skip number 5 in 1-10
for (int i = 1; i <= 10; i++) {
    if (i == 5) continue;
    System.out.print(i + " ");
}
// Output: 1 2 3 4 6 7 8 9 10  (5 is skipped)

// Example 3: Sum positive numbers only
int[] data = {5, -3, 8, -1, 4, -7, 2};
int sum = 0;
for (int val : data) {
    if (val < 0) continue;  // skip negative numbers
    sum += val;
}
System.out.println("Sum of positives: " + sum);  // 19
break vs continue comparison: for (i=1; i<=5; i++) { for (i=1; i<=5; i++) { if (i==3) BREAK; if (i==3) CONTINUE; print(i); print(i); } } Output: 1 2 Output: 1 2 4 5 ↑ ↑ stops at 3 skips 3, resumes at 4

Labeled Loops — break/continue Outer Loop

Labeled break — Exit Outer Loop from Inner
// Without label: break exits INNER loop only
// With label: break exits the NAMED loop

outer:  // label — can be any name
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        if (i == 2 && j == 2) {
            break outer;  // exits BOTH loops
        }
        System.out.print("("+i+","+j+") ");
    }
}
// Output: (1,1) (1,2) (1,3) (2,1)
// Stops when i=2,j=2 — both loops exited

8.8 — ModernEnhanced for Loop (for-each)

The enhanced for loop (also called for-each) provides a cleaner syntax for iterating over arrays and collections. It eliminates index management entirely — use it when you need to read every element but don't need the index.

Enhanced for — for-each Loop
int[] scores = {85, 92, 78, 96, 88};

// Regular for (need index):
for (int i = 0; i < scores.length; i++) {
    System.out.print(scores[i] + " ");
}

// Enhanced for (cleaner, no index):
for (int score : scores) {       // "for each score in scores"
    System.out.print(score + " ");
}
// Output: 85 92 78 96 88

// Computing sum with enhanced for:
int sum = 0;
for (int score : scores) sum += score;
double avg = (double) sum / scores.length;

// Works with String arrays too:
String[] names = {"Ali", "Sara", "Omar"};
for (String name : names) {
    System.out.println("Hello, " + name);
}

// LIMITATION: cannot modify array elements (reads only)
for (int score : scores) {
    score = score + 10;  // modifies local copy, NOT the array!
}
// Use regular indexed for if you need to modify elements

8.9 — ProgramsClassic Loop Problems

Factorial

Factorial — n!
import java.util.Scanner;
public class Factorial {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter n: ");
        int n = sc.nextInt();

        long factorial = 1;  // use long — factorials get big fast!
        for (int i = 1; i <= n; i++) {
            factorial *= i;
        }
        System.out.println(n + "! = " + factorial);
        sc.close();
    }
}
// n=5: 1×2×3×4×5 = 120
// n=0: factorial = 1 (loop never runs, correct — 0! = 1)

Prime Number Checker

Prime Checker — Optimised
int num = 29;
boolean isPrime = true;

if (num < 2) {
    isPrime = false;
} else {
    // Only need to check divisors up to sqrt(num)
    for (int i = 2; i <= Math.sqrt(num); i++) {
        if (num % i == 0) {
            isPrime = false;
            break;  // found a factor — no need to continue
        }
    }
}
System.out.println(num + (isPrime ? " is prime" : " is not prime"));
// 29 is prime

Number Reversal and Digit Sum

Reverse a Number + Digit Sum
int n = 12345;
int reversed = 0, digitSum = 0;
int temp = n;  // preserve original

while (temp > 0) {
    int lastDigit = temp % 10;   // extract last digit
    digitSum += lastDigit;         // add to sum
    reversed = reversed * 10 + lastDigit;  // build reversed
    temp /= 10;                   // remove last digit
}

System.out.println("Original:  " + n);          // 12345
System.out.println("Reversed:  " + reversed);   // 54321
System.out.println("Digit sum: " + digitSum);   // 15

8.10 — Watch OutMost Common Loop Mistakes
Top 8 Loop Mistakes
  1. Off-by-one error — Loop runs one time too many or too few. Check: should condition be < or <=? Should i start at 0 or 1? Think through the first and last iteration manually.
  2. Infinite loop — Condition never becomes false. Common causes: forgetting to increment i, wrong condition direction, modifying the wrong variable.
  3. Semicolon after for/while — for (int i=0; i<5; i++); — the empty statement is the entire loop body. The block after it runs once, after the loop ends.
  4. Using wrong variable in nested loops — Using outer i where inner j is needed inside a nested loop. Carefully name variables.
  5. Modifying loop variable inside body — for(int i=0; i<n; i++){ i+=2; } — the update in the header AND inside body both change i. Unpredictable.
  6. Using enhanced for to modify array — for(int x : arr) { x = x*2; } — this modifies the local copy x, NOT the array element. Use indexed for to modify.
  7. Forgetting println() after inner loop — In nested loops for patterns, the newline must go inside the outer loop but OUTSIDE the inner loop.
  8. Integer overflow in accumulator — Using int for factorial or sum of large numbers. Use long to avoid silent overflow.
Semicolon Bug + Off-by-One
// BUG 1: Semicolon after for — empty loop body
for (int i = 0; i < 5; i++);   // loops 5 times doing NOTHING
System.out.println("Only once!"); // runs once AFTER loop

// BUG 2: Off by one — should be <= but uses <
for (int i = 1; i < 10; i++)     // prints 1-9 (misses 10!)
    System.out.print(i + " ");

// BUG 3: Infinite loop — forgot increment
int i = 0;
while (i < 5) {
    System.out.println(i);
    // i++ MISSING — loops forever!
}

Memory Tips — Chapter 8
  • for vs while: "for = known count (FOR a set number of times). while = unknown count (WHILE a condition holds)."
  • do-while guarantee: "do-while is the 'at least once' loop — like a restaurant that makes you try a dish at least once."
  • break vs continue: "break = break out of the loop entirely. continue = continue to the next iteration (skip the rest of this one)."
  • Off-by-one check: Always trace the first and last iteration mentally. Ask: "Does i=1 work? Does the last valid value get included?"
  • Enhanced for limitation: "for-each reads, indexed for writes. Use for-each when you only need to READ array elements."
  • Semicolon bug: Same as if — a semicolon after for() or while() creates an empty loop body. Never put ; right after the closing parenthesis of a loop header.

8.12 — ExamUniversity Exam Preparation
3 Marks — Compare

Q: Compare while and do-while loops with examples. When would you specifically choose do-while over while?

Answer: Both while and do-while repeat based on a condition, but differ in when they check it. while checks the condition BEFORE the body — if the condition is false initially, the body never executes (can run 0 times). do-while checks the condition AFTER the body — the body always executes at least once regardless of the condition. Use do-while specifically when the loop body must execute at least once: most common for input validation (show a prompt at least once, keep asking until valid input) and menu-driven programs (show the menu at least once, repeat while user doesn't choose exit).

4 Marks — Write a Program

Q: Write a Java program to check whether a number is an Armstrong number. (An Armstrong number equals the sum of its digits each raised to the power of the number of digits. e.g., 153 = 1³ + 5³ + 3³)

Model Answer
import java.util.Scanner;
public class Armstrong {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter number: ");
        int n = sc.nextInt();
        int temp = n, digits = 0, sum = 0;
        // Count digits
        while (temp > 0) { digits++; temp /= 10; }
        // Calculate sum of digit^digits
        temp = n;
        while (temp > 0) {
            int d = temp % 10;
            sum += (int) Math.pow(d, digits);
            temp /= 10;
        }
        System.out.println(n + (sum == n ? " IS" : " is NOT") + " an Armstrong number");
        sc.close();
    }
}

Quick Revision — Chapter 8
Three loops: for (known count), while (condition-based, 0+ runs), do-while (always 1+ runs).
for order: Init(once) → [Condition → Body → Update] → repeat → Condition(false) → done.
while: check BEFORE body. do-while: check AFTER body — body always runs at least once.
Nested loops: inner runs completely for each outer iteration. Total = outer × inner iterations.
break: exits loop entirely. continue: skips rest of current iteration, moves to next.
Labeled break/continue: outer: for{...} → break outer; exits the labeled loop.
Enhanced for: for(type var : array) — reads elements cleanly. Cannot modify array elements.
Common bugs: off-by-one (< vs <=), infinite loop (missing update), semicolon after for/while.
Digit ops pattern: lastDigit = n%10; n/=10; in a while loop to process each digit.
